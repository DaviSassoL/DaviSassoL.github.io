Pipeline de Dados & IA para Treinos Personalizados

Projeto de arquitetura de dados para consolidar informações de treinos físicos, integrando API Hevy, formulários de usuários e datasets externos em um Data Lake na AWS. A solução utiliza Airflow para orquestrar o pipeline Bronze–Silver–Gold e um app Streamlit com IA para apoiar praticantes e instrutores na criação de treinos personalizados baseados em dados.

Contexto

Setor & Problema de Negócio

Setor – Saúde, Fitness & Bem-estar
  • Crescimento de apps de treino e plataformas digitais de bem-estar.
  • Usuários registram treinos em múltiplos sistemas, nem sempre integrados.
  • Personal trainers precisam de visão consolidada para ajustar planos.
Problema de Negócio
  • Ausência de visão unificada de treinos, cargas e volume ao longo do tempo.
  • Dificuldade para comparar evolução individual com médias globais.
  • Tomada de decisão pouco orientada a dados na prescrição de treinos.

Pergunta central: como usar Engenharia de Dados para transformar dados de treino em insights acionáveis para usuários e instrutores?

Implementar uma arquitetura de dados em nuvem, inspirada no modelo Lambda, para suportar um MVP analítico de treinos personalizados, com Data Lake em S3, pipelines orquestrados por Airflow e um app Streamlit com IA.

  • Suportar recomendações de treino personalizadas com IA.

Slide 5 · Arquitetura Lambda Próximo: Bronze / Silver / Gold
Data Lake em AWS S3

Camadas Bronze, Silver e Gold

Bronze – Raw
  • Dados brutos exatamente como chegam das fontes.
  • Ex.: hevy/raw_workouts.parquet, users/raw_form.parquet.
  • Foco em rastreabilidade e histórico completo.
Silver – Trusted
  • Dados limpos, tipados e integrados.
  • Tabelas de usuários, treinos, séries e exercícios.
  • Aplicação de regras de qualidade, conversão de tipos e cálculo de campos derivados.
Gold – Curated
  • Modelagem em Star Schema com tabelão de métricas.
  • fact_workout_metrics, dim_user, dim_date, dim_exercise.
  • Base otimizada para dashboards e análises OLAP / ML.

Decisão de design: usar um Star Schema na Gold para simplificar consultas analíticas e manter flexibilidade para novos indicadores.

Slide 6 · Camadas do Data Lake Próximo: Orquestração com Airflow
Orquestração

Airflow como maestro do pipeline

O Apache Airflow coordena todas as etapas do ETL, garantindo dependências, agendamentos e reexecução em caso de falhas. Em vez de scripts soltos, o pipeline passa a ser descrito como um grafo de tarefas.

DAG principal
  • create_lake_structure – prepara a estrutura do Data Lake em S3.
  • ingest_users_form, ingest_hevy, ingest_kaggle – ingestão paralela para Bronze.
  • silver_users, silver_transform, silver_kaggle – transformações para Silver.
  • gold_metrics – consolidação final na camada Gold.

O Airflow executa as tasks em ordem, monitora logs, permite retries automáticos e facilita o reprocessamento de períodos específicos quando necessário.

Na prática
  • Cada task usa BashOperator para disparar scripts Python.
  • Execuções podem ser agendadas (ex.: diária) ou disparadas manualmente.
  • Falhas interrompem as tasks dependentes, evitando dados inconsistentes na Gold.
  • A interface do Airflow exibe o grafo, status de cada task e histórico de runs.
Slide 7 · Airflow Próximo: Modelagem de Dados
Modelagem de Dados

Star Schema na camada Gold

Tabela Fato – fact_workout_metrics
  • Grão: métricas de treino por usuário, data e exercício.
  • Métricas: total de séries, repetições, volume (kg), carga máxima, frequência.
  • Chaves: date_key, user_key, exercise_key.
Dimensões Principais
  • dim_user: dados de perfil, objetivos, experiência.
  • dim_date: dia, semana, mês, ano, dia da semana, feriados.
  • dim_exercise: nome, grupo muscular, tipo de equipamento.
Benefícios
  • Facilidade de leitura por ferramentas de BI.
  • Consultas analíticas simples com agregações por tempo, usuário ou exercício.
  • Base consistente para futuros modelos de Machine Learning.

A opção por um Star Schema equilibra performance de consulta, simplicidade de entendimento e flexibilidade para novos indicadores de treino.

Slide 8 · Modelagem Star Schema Próximo: MVP com Streamlit & IA
MVP · Aplicação

Streamlit App & Chat IA

A camada de consumo materializa o valor do pipeline para o usuário final, através de um app web simples, porém conectado ao Data Lake e a um modelo de IA.

Streamlit
  • Login com usuários armazenados em users_app.parquet no S3.
  • Formulário coleta dados de perfil e objetivos, gravados em user_profiles/.
  • Dashboard lê a camada Gold para exibir evolução de treinos e comparações.
Chat IA – Personal Trainer
  • Integração com modelo LLaMA 3.1 via Groq API.
  • Uso de contexto: perfil + histórico de treinos do usuário.
  • Gera recomendações de treino e ajustes semanais personalizados.
Dados de teste
• Script generate_fake_test_data.py gera fact_workouts_test_user.parquet na área de teste do S3.{"\n"} • Usuário de demonstração: testuser@example.com / 123456.{"\n"} • Permite validar toda a jornada sem depender de APIs externas em tempo real.
Slide 9 · MVP Streamlit & IA Próximo: Governança & Qualidade
Boas Práticas

Governança, Qualidade & Automação

Governança & LGPD
  • Nomenclatura por camada/domínio: bronze/hevy, silver/users, gold/fact_workout_metrics.
  • Uso mínimo de dados pessoais; foco em identificadores técnicos.
  • Possibilidade de criptografia em repouso (S3) e controle de acesso via IAM.
Qualidade de Dados
  • Validação de tipos, faixas e nulos nas transformações Silver.
  • Deduplicação de registros por chaves de negócio + timestamps.
  • Cálculo de métricas derivadas (volume, cargas) de forma padronizada.
CI/CD & IaC (Visão Conceitual)
  • GitHub como repositório central e controle de versão.
  • Pipeline CI/CD proposto com GitHub Actions para testes e build de containers.
  • Terraform sugerido para provisionar bucket S3, usuários IAM e infraestrutura de execução.

Encerramento

Resumo & Agradecimentos

O projeto Instructor Workout demonstra, de forma integrada, como Engenharia de Dados, nuvem e IA podem apoiar decisões no contexto de treinos físicos. A solução conecta múltiplas fontes de dados, organiza-as em um Data Lake estruturado e disponibiliza insights em um MVP acessível.

Pontos-chave
  • Arquitetura Lambda adaptada ao contexto de fitness.
  • Data Lake em AWS com Bronze, Silver e Gold.
  • Airflow garantindo orquestração e reprocessamento.
  • Modelagem Star Schema para análise de treinos.
  • App Streamlit + IA aproximando o usuário final dos dados.
Aprendizados
  • Importância de desenhar camadas de dados desde o início.
  • Benefícios de separar ingestão, transformação e consumo.
  • Como pequenos incrementos (MVP) aceleram validação de ideias.
