Contexto & Motivação

Por que construir uma plataforma de treinos orientada a dados?
Problema Real de Negócio

Cenário Atual

Apps de treino ainda são genéricos

  • Planos de treino pouco adaptados ao comportamento real do aluno.
  • Falta de visão consolidada da evolução física ao longo do tempo.
  • Dados de treinos espalhados, pouco estruturados e pouco usados.

Oportunidade: usar Engenharia de Dados + Analytics para transformar dados de treino em um motor de personalização.

Proposta

  • Capturar treinos reais via Hevy API.
  • Centralizar tudo em um Data Lake escalável.
  • Enriquecer os dados com métricas analíticas e comportamentais.
  • Preparar a base para dashboards e modelos de IA.

Objetivo do MVP

Validar o pipeline de dados e a arquitetura da solução.
Versão 1.0 da Plataforma

O que queremos provar

  • Conseguir ingerir treinos reais de forma incremental e confiável.
  • Organizar os dados em Bronze, Silver e Gold (Medallion Architecture).
  • Entregar dados prontos para dashboards e experimentos de Machine Learning.

Resultados Esperados

  • Data Lake funcional com histórico de treinos.
  • Modelo dimensional para análise da performance dos alunos.
  • Wide table com features para futuros modelos de recomendação.

Plataforma Inteligente

Resumo

  • O MVP comprova a aplicação prática de Engenharia de Dados no contexto de treinos físicos.
  • A arquitetura em camadas (Bronze, Silver, Gold) garante histórico, qualidade e escalabilidade.
  • O ambiente está pronto para evoluir com dashboards avançados e modelos de Machine Learning.

Próximo nível

  • Transformar a base Gold em recomendações reais de treino.
  • Fechar o ciclo entre dados, instrutor e aluno.
